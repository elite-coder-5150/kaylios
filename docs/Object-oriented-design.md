## The four fundamental principles of object oriented programing are:



1.   **Abstraction** - Abstraction is the process of simplifying the complex system by 

     focusing on the essential features. In OOP, this refers to the idea of creating 

     abstract classes or interface that defined a set of methods that can be implemented

     by different classes. Abstraction helps hide implementation details and promotes 

     modular design

     

2.   **Encapsulation** - Encapsulation is the idea of bundling data and functions that operate on that ata within a single unit called a class. The data is not accessible to the outside world and can only be accessed through the function or methods that you deined within the class

     

3.   **Inheritance** - Inheritance is the process by which a class can be derived from another class, inheriting its properties and methods. The new class is called the subclass or derived class, and the original class is the superclass or the base class. Inheritance allows for code reuse and promote a hierarchical structure.

     

4.   **Polymorphism** - Polymorphism means the abolity for an object to take on many forms. In OOP, this refers to the ability of objects of different classes to be used interchangeably. This is acheived throught method overloading, which alls different methods to have the same name but different parameters, and method overriding, which allows a subclass to provide a different implementation of a method taht is already defined in its superclass.
